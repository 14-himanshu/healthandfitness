# Vitality AI - Clinical-Grade Health Assistant

Vitality is an advanced, decoupled AI Health and Fitness ecosystem designed to provide clinical-grade, fact-checked, and highly personalized health insights.

Unlike standard fitness chatbots, Vitality utilizes a dual-agent architecture with a built-in Medical Safety Evaluator, Retrieval-Augmented Generation (RAG) for citing real medical journals (PubMed), live nutritional database integration (USDA), and a permanent clinical memory store.

## 📂 Project Structure

The platform is decoupled into three distinct microservices to ensure scalability, security, and separation of concerns.

```text
healthandfitness/
├── frontend/             # The UI (React + Vite)
│   ├── src/              # React components (e.g. AICopilot with Rich Agent UI)
│   └── public/           # Static assets (Favicon)
├── backend/              # The API Proxy & DB (Node.js + Express + SQLite)
│   ├── config/           # SQLite database configuration (metrics, goals, clinical_data)
│   ├── controllers/      # Handles PII scrubbing and forwards requests to the Python agent
│   └── routes/           # Express routes
└── python_agent/         # The AI "Brain" (FastAPI + Groq + ChromaDB)
    ├── agent.py          # Dual-Agent logic, RAG ingestion, and Memory injection
    ├── usda_api.py       # Fetches live USDA FoodData Central nutritional facts
    └── pubmed_api.py     # Fetches real NCBI clinical studies for fact-checking
```

## 🚀 Key Features

1. **Dual-Agent Medical Triage:** Every output generated by the primary LLM is secretly intercepted by an "Evaluator LLM" which checks for extreme calorie deficits or medical emergencies. If a user complains of chest pain, the agent overrides the response with an `EMERGENCY_TRIAGE` UI alert.
2. **Persistent Clinical Memory:** The Node backend actively pulls a user's SQLite history (weight, daily calories, and past lab results like "LDL cholesterol") and injects it into the agent's prompt so it has long-term memory of the user's health trajectory.
3. **Live Fact-Checking (RAG):** When asked about supplements or medical queries, the agent queries the PubMed API, retrieves recent clinical studies, and outputs answers with explicit citations.
4. **PII Scrubbing:** Before the Node.js backend sends any user context to the external Python AI, it dynamically scrubs personal identifiers (emails, phone numbers) to ensure zero-data-retention compliance.
5. **Rich UI Interactions:** The React frontend parses the specific `actionType` returned by the agent and renders beautiful, native components (e.g. Green Nutritional Fact sheets, Red Medical Sirens, Blue PubMed Citations) rather than generic text.

## 🛠️ How to Run Locally

Because of the decoupled architecture, you need to run all three services simultaneously.

### 1. Start the Python AI Agent
```bash
cd python_agent
source venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 2. Start the Node.js Backend
```bash
cd backend
npm install
npm run dev
```
*(Runs on `http://localhost:4000`)*

### 3. Start the React Frontend
```bash
cd frontend
npm install
npm run dev
```
*(Runs on `http://localhost:5173`)*

## 🔑 Environment Variables
You will need valid API keys for the Python agent to pull live data. Create a `.env` in the `python_agent` folder (or ensure they are hardcoded for testing as we did during the build phase):
* `GROQ_API_KEY`: To run the Llama-3 models.
* `USDA_API_KEY`: To fetch nutritional data.
* `PUBMED_API_KEY`: To fetch medical studies via NCBI E-utilities.
